# نظام الكنترول المتكامل - v2.2.0

## هيكل المشروع

```
kontrolSystem/
│
├── index.html              ← الصفحة الرئيسية (HTML فقط - بلا JavaScript)
│
├── css/
│   └── main.css            ← جميع أنماط CSS
│
├── js/
│   ├── settings.js         ← إدارة الإعدادات (localStorage + مزامنة الواجهة)
│   ├── utils.js            ← الدوال المساعدة (readExcel، parseGrade، applyRaise...)
│   ├── table.js            ← عرض الجداول + البحث + الطباعة
│   ├── stats.js            ← الإحصائيات والرسم البياني (Chart.js)
│   ├── normalSystem.js     ← النظام العادي + قاعدة H + الموافقة
│   ├── flexSystem.js       ← النظام المرن (القواعد 1، 2، 3)
│   └── app.js              ← نقطة البداية - تشغيل كل الأنظمة
│
└── logos/
    ├── ministry-logo.png   ← شعار الوزارة
    └── institute-logo.png  ← شعار المعهد
```

## ترتيب تحميل JavaScript

الملفات تُحمَّل بهذا الترتيب (مهم):
1. `settings.js`     → المتغيرات والإعدادات العامة
2. `utils.js`        → الدوال المساعدة
3. `table.js`        → عرض الجداول
4. `stats.js`        → الإحصائيات
5. `normalSystem.js` → النظام العادي
6. `flexSystem.js`   → النظام المرن
7. `app.js`          → التشغيل النهائي

## ميزة المسح التلقائي (جديد v2.2.0)

عند رفع أي ملف جديد (طلاب أو موافقة) يتم **تلقائياً**:
- مسح جدول النتائج السابق
- مسح رسائل المعالجة
- مسح إحصائيات المقارنة والرسم البياني
- مسح قائمة المستفيدين

هذا يضمن عدم ظهور بيانات قديمة مع ملف جديد.

## قواعد النظام المرن

| القاعدة | الوصف |
|---------|-------|
| القاعدة 1 | تحويل درجة النموذج إلى ميد+أعمال عبر الشرائح |
| القاعدة 2 | إضافة 20 إلى أعمال السنة (بحد أقصى 30) + الميد والفاينل كما هو |
| القاعدة 3 | توزيع 35 على الميد والأعمال (ثابت أو تصاعدي حتى 50) |

## قاعدة H

للطلاب **غير** الموافقة وغير الغائبين:
```
أعمال_جديد = ROUNDUP(أعمال + H + (فاينل/50 × H) , 0)
الحد الأقصى = 30
```

## طلاب الموافقة (جميع الأنظمة)

يُحسب الميد والأعمال وفق الفاينل:
```
ميد_جديد = 15 + (فاينل/50) × معامل_الميد     [بين 10 و 20]
أعمال_جديد = 20 + (فاينل/50) × معامل_الأعمال  [بين 10 و 30]
```
